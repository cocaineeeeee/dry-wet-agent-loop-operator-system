# expos 路线图 · 生物成为主研究方向（用户正式裁决 2026-07-14，权威）

> **战略转向**：生物不是第三个 demo，而是 expos 接下来的主研究方向。化学两域
> （solvent/catalyst）已完成最重要任务——证明 runtime 可换域；再做更多化学变体
> 边际价值已低。生物才真正逼系统面对新科学物件（molecule/reaction → sequence/
> construct → expression/activity/phenotype）。repo 取证支持此向：kernel/claim
> ledger/knowledge compiler/主统计裁决**未被生物逼改**，暴露的问题集中在 provider
> contract 与 MCL dry orchestration——核心有资格继续走向 biology，外层契约为此进化。

## 主线（科学）与支线（工程）

```
科学主线：biology          ← 主体
工程支线：real-wet hardware ← 平行、降级，不取消
```

接一台机器只证明 expos 能控制设备；跑通 sequence→phenotype→knowledge→redesign 才
证明它能参与生命科学研究。

## M24-A：打通生物执行面（前置工程，非最终成果）

## M24-B：完成真正的生物闭环（最小但完整）

8-16 constructs / 2-4 rounds / 4-8 replicates / 单 fluorescence observable / 三判别面。
**必须出现**：knowledge fingerprint migration + proposal-order change + claim
supported/rejected/insufficient 分离。

## M25：从筛选走向设计

parent→variant lineage / sequence mutation operators / diversity-aware acquisition /
multi-feature construct design / ESM·RNA foundation model 冻结推理。expos 从
"screen biological candidates" 升为 "generate and test biological designs"。

## 成功判准（第一个生物 loop——非"bio domain loads"）

真正交付 = **完整的、provenance-aware 的自适应生物 loop：实验表型数据改变科学知识，
被改变的知识改变下一轮 construct 设计**。六件同时成立：
1. sequence construct 无需伪造分子几何；
2. 生物数据真的改变 claim；
3. claim 改变 knowledge；
4. knowledge 真的改变下一轮候选；
5. high/flipped/flat 三判别面全成立；
6. kernel/ledger/certification 保持生物盲。

## 反模式（明令避免）

- 停在零件：「又支持一个 YAML 域 / 算了几个 GC-CAI / 模拟了 fluorescence / 接了 ESM」；
- 立即扩张：single-cell / omics / 显微 / 细胞培养 / 药物发现 / 大型蛋白 LM 训练——
  会让 domain/模型/数据/仪器/统计五问题同时混一起。
第一个生物 loop 必须**小，但科学因果链完整**。

## 用户正式裁决原文（可直接传两会话）

```
User ruling — Biology becomes the primary scientific direction of expos.

Do not treat cell_free_expression_screen as merely a third-domain demo.
The objective is to evolve expos from a chemistry-capable runtime into a
biological adaptive research system.

Priority order:
1. Complete only the minimum Domain Contract v3 and dry-adapter injection
   needed to represent biological constructs honestly.
2. Build and close an adaptive cell-free expression loop.
3. Demonstrate sequence → phenotype → evidence → claim → knowledge →
   redesigned sequence selection.
4. Add generative sequence optimization only after the fixed-pool biological
   loop is countersigned.
5. Keep real-hardware readiness as a parallel engineering track, not the main
   scientific objective.

Hard requirements:
- kernel, planner, evidence compiler, ledger, and knowledge compiler remain
  biology-agnostic;
- no fake Z-matrix or chemistry-shaped placeholder for biological constructs;
- biological terminology and policies live only in domain/provider/adapter/QC
  layers;
- technical replicates are not counted as independent biological evidence;
- positive, negative, and reference controls are mandatory;
- normalization and batch correction happen before certification;
- public biological datasets may calibrate priors and simulators, but may not
  be presented as observations generated by the current run;
- high, flipped, and flat acceptance faces are required;
- success requires a measured knowledge-fingerprint change and a subsequent
  proposal change.

The first major deliverable is not "bio domain loads." It is:
A complete, provenance-aware adaptive biological loop in which experimental
phenotype data changes scientific knowledge and that changed knowledge alters
the next construct design.
```
