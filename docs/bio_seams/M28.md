# M28 biology_discovery — seam list (v0.1 skeleton)

Team M28 "Understand". Skeleton delivered: `agents/biology_discovery/` with four
deterministic agent stubs (Hypothesis / Analysis / Contradiction / Replication) +
objects. Smoke green: `python -m agents.biology_discovery` (≥2 competing hypotheses →
contradiction detected → trusted+replicated evidence → up=SUPPORTED / down=REJECTED
separation; untrusted same evidence → both INSUFFICIENT). Reuses `expos.kernel.claims.
ClaimDecisionStatus` READ-ONLY (kernel unchanged). No LLM (later seam).

Seams for the integration owner (B):

1. **claim lifecycle** — agents currently PROPOSE verdicts (`AnalysisVerdict`) but do NOT
   mutate the ledger. Seam: route an analysis verdict through the *existing* certification
   → `ClaimDelta` → `evaluate_gate` path so only trusted evidence mutates a claim head.
   The red line (untrusted ⇒ INSUFFICIENT, non-mutating) is already enforced agent-side;
   the kernel gate must remain the sole mutator. NO biological specialization in the gate.
2. **trusted-evidence source** — `Evidence.trusted` must be fed from the wet/certified
   observation stream (an adjudicated `Observation`), not agent-authored. Seam: an adapter
   from a certified round observation → `Evidence(trusted=True)`.
3. **hypothesis object** — `Hypothesis` (perturbation, axis, direction) should map onto the
   shared `HypothesisObject` mcl already threads (`_domain_hypotheses`), so posed
   hypotheses ride the same knowledge substrate. Seam: a converter + a stable hypothesis id
   scheme.
4. **LLM proposer/critic** — the four stubs are pure functions today; the LLM backend
   (proposer for HypothesisAgent, critic for ContradictionAgent) is a dependency-injected
   seam (mirror `agent_backend` injection in `run_mcl_loop`), default template/deterministic.
5. **replication policy** — `ReplicationAgent(min_reps)` is a local constant; it should read
   the domain's `replicate_kind` (biological vs technical) so replication counts match the
   certification's independence assumption.
6. **fingerprints** — analysis is deterministic; the agent-set version + decision-fn id
   should enter provenance so a verdict is reproducible from the event stream.
