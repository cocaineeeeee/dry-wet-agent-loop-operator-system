## What is now delivered (domain-local, runnable)

```
python -m domains.generative_construct              # full v0.1 story, deterministic
python -m pytest tests/test_m25_generative_v01.py   # 24 tests
```

- **Five auditable operators** (`expos/adapters/dry/mutation_operators.py`) ‚Äî pure
  `apply(parent, op_params, seed) -> (child_components, EditProvenance)`:
  `promoter_swap` ¬∑ `rbs_swap` ¬∑ `codon_optimize` ¬∑ `utr5_mutation` ¬∑
  `cds_synonymous`. Deterministic; `sequence == promoter+rbs+cds` invariant; the
  two CDS operators are **translation-invariant** (synonymous, dual-use-safe).
  `child_id` is content-addressed and never reuses the parent/observation id.
  `module_fingerprint()` exposes the source hash for run identity.
- **Generation pool** (`domains/generative_construct/generation.py`) ‚Äî parent‚ÜíN
  children over a 5-operator grid; `fixed_pool_seed_lineage()` = the 11-construct
  canary.
- **Append-only lineage + PROV** (`objects.py`) ‚Äî `DesignLineage` refuses in-place
  rewrites; each edit lands a PROV activity (`used_entity`=parent,
  `generated_entity`=child, `operator_fingerprint`).
- **Diversity-aware acquisition** (`acquisition.py`) ‚Äî observation-independent,
  deterministic; onehot/composition encoding + GREEDY / DIVERSITY / MMR
  (`value_diversity`). ADAPTed from ALDE's score-pool‚Üígreedy-pick‚Üímask grid.
- **Domain-local wet + faces** (`wet_truth.py`) ‚Äî SIMULATED phenotype reusing the
  shared `TruthSurface` (imported, not edited). Faces: `expression_high`
  (positive), `expression_flipped` (negative), `flat` (null).
- **Domain-local e2e** (`e2e.py`) ‚Äî closes obs‚Üíclaim‚Üíknowledge‚Üínext-decision on
  the **existing** `kernel.claims` ledger + `kernel.knowledge` compiler. Three
  acceptance faces verified: positive‚ÜíSUPPORTED, overturn‚ÜíREJECTED, flat‚Üí
  INSUFFICIENT; a REJECTED overturn re-steers the next round's acquisition
  (`greedy`‚Üí`value_diversity`).
- **Headline discriminator**: `provider.discriminative_pool()` ‚Äî a strong RBS +
  optimal CDS grafted onto a transcriptionally-weak (GC-rich) promoter scores
  HIGH on the dry proxy but LOW in simulated wet (the proxy is blind to promoter
  functional strength). Dry ranking is overturned by wet phenotype.

## Seams still owned by the integration owner (B, single-writer of mcl / schema)

1. **candidate-source hook** ‚Äî let `GenerativeConstructProvider.propose_children`
   feed *generated* (not just preset-catalogue) candidates into an mcl round's
   candidate pool. Needs a `provider.propose_candidates(round, knowledge)`
   entrypoint on the shared provider protocol. Generated child components are the
   same `{sequence,promoter,rbs,cds}` `sequence_construct` kind M24 already routes
   ‚Äî **no new input_kind**.
2. **acquisition ‚Üí policy** ‚Äî wire `provider.acquire(pool, k, strategy)` (returns
   ranked `Selection`s with value/diversity/acq_score) into the mcl selection
   policy so a round screens the diversity-aware batch. The strategy chosen can be
   knowledge-driven (see #4).
3. **lineage provenance ‚Üí DesignProvenance** ‚Äî fold `EditProvenance.as_activity()`
   (operator id + params + `operator_fingerprint`, `generator="generative_
   construct/v0.1"`) into the experiment's `DesignProvenance` so a generated
   candidate's ancestry is auditable and `generator != mcl_template_agent`.
4. **claim + knowledge wiring** ‚Äî the domain-local e2e already builds real
   `ClaimDelta`s (decision_fn `m25_dry_ranking_verdict` v1, registered in the
   shared `DECISION_FN_REGISTRY`) and compiles a `KnowledgeView`. Seam: emit the
   `claim_decision` / `knowledge_updated` events at mcl round-end (the emit
   helpers already exist) and let the compiled knowledge pick the next round's
   acquisition strategy through mcl (the e2e computes `next_strategy` locally).
5. **fingerprints** ‚Äî fold `mutation_operators.module_fingerprint()` into
   `config_fingerprint` when generated candidates are used, so a change to any
   operator flips run identity (like the dry-adapter source hash).
6. **real wet path** ‚Äî `wet_truth.wet_phenotype` is a domain-local SIMULATION. The
   real path replaces it with the socket `sim_reader` truth face already carrying
   `expression_high` / `expression_flipped` / `flat` (no sim_reader change needed
   ‚Äî the faces exist). Honest label stays `simulation` until real hardware.
7. **QC trust gate** ‚Äî the e2e asserts observation trust by construction. Real
   wiring routes the simulated readings through the existing wet QC / trusted-
   observation path before they can certify.

## Honest limits (v0.1)
`simulation` level (not retrospective, not wet). The verdict statistic is a simple
mean-difference band ‚Äî a real permutation test + evidence factor is K-B's job. No
PLM / large generative model (explicit v2 seam in `sequences.expression_features`
and the `acquisition` UCB/TS GP-posterior column). kernel/ledger/evidence-compiler/
knowledge-compiler carry **zero** M25 specialization.
