### B's two fold-in points (now ready on the provider)

1. **`propose_candidates` → candidate pool** (seam #1 below, now concrete). The loadable
   provider exposes
   `propose_candidates(parent_target_id, *, seed=0, parent_components=None) ->
   Mapping[str, ComputeTarget]`: it expands a parent level into the five-operator CHILD
   pool (delegating to `domains.generative_construct.generation.generate_children`) and
   returns child_id → `sequence_construct` ComputeTarget. **Fold:** each mcl round, call
   it on the current parent(s) and MERGE the returned children into the round's candidate
   pool (they route to the SAME `SequenceProxyAdapter` as the preset base — no new
   `input_kind`). `parent_target_id` defaults to a preset (lineage root); pass
   `parent_components` to grow deeper generations. Children are content-addressed and
   never reuse the parent/observation id.
2. **operator hash → `config_fingerprint`** (seam #5 below, now concrete). The provider
   exposes `operator_fingerprint() -> "sha256:…"` (= `mutation_operators.
   module_fingerprint()`). **Fold:** WHEN a round uses generated candidates, fold this
   into `config_fingerprint` (alongside the provider's own `provider_fingerprint`) so a
   change to any of the five operators flips run identity — the dry-adapter-source-hash
   precedent. It is a SEPARATE fold from `provider_fingerprint` (operators live in
   `adapters/dry/mutation_operators.py`, imported by reference).

- **Five auditable operators** (`expos/adapters/dry/mutation_operators.py`) — pure
  `apply(parent, op_params, seed) -> (child_components, EditProvenance)`:
  `promoter_swap` · `rbs_swap` · `codon_optimize` · `utr5_mutation` ·
  `cds_synonymous`. Deterministic; `sequence == promoter+rbs+cds` invariant; the
  two CDS operators are **translation-invariant** (synonymous, dual-use-safe).
  `child_id` is content-addressed and never reuses the parent/observation id.
  `module_fingerprint()` exposes the source hash for run identity.
- **Generation pool** (`domains/generative_construct/generation.py`) — parent→N
  children over a 5-operator grid; `fixed_pool_seed_lineage()` = the 11-construct
  canary.
- **Append-only lineage + PROV** (`objects.py`) — `DesignLineage` refuses in-place
  rewrites; each edit lands a PROV activity (`used_entity`=parent,
  `generated_entity`=child, `operator_fingerprint`).
- **Diversity-aware acquisition** (`acquisition.py`) — observation-independent,
  deterministic; onehot/composition encoding + GREEDY / DIVERSITY / MMR
  (`value_diversity`). ADAPTed from ALDE's score-pool→greedy-pick→mask grid.
- **Domain-local wet + faces** (`wet_truth.py`) — SIMULATED phenotype reusing the
  shared `TruthSurface` (imported, not edited). Faces: `expression_high`
  (positive), `expression_flipped` (negative), `flat` (null).
- **Domain-local e2e** (`e2e.py`) — closes obs→claim→knowledge→next-decision on
  the **existing** `kernel.claims` ledger + `kernel.knowledge` compiler. Three
  acceptance faces verified: positive→SUPPORTED, overturn→REJECTED, flat→
  INSUFFICIENT; a REJECTED overturn re-steers the next round's acquisition
  (`greedy`→`value_diversity`).
- **Headline discriminator**: `provider.discriminative_pool()` — a strong RBS +
  optimal CDS grafted onto a transcriptionally-weak (GC-rich) promoter scores
  HIGH on the dry proxy but LOW in simulated wet (the proxy is blind to promoter
  functional strength). Dry ranking is overturned by wet phenotype.
