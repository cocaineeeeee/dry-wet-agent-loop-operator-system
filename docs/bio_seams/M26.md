## SEAM 2 — time-series observable schema (**LANDED; schema updated this pass**)
**Shared files:** `expos/adapters/wet/sim_reader.py` (or a new neutral observation carrier), the
observation-ingest path, `expos/kernel/objects.py` observation shape (READ-ONLY confirm neutrality).

  reading. M26's convention (kernel-neutral, mirrors the M24 value/secondary split):
  `RawResult.value` = the load-bearing dynamic scalar (cassette steady-state / toggle bistable
  separation / oscillator frequency); `RawResult.secondary` = the fixed derived-phase bundle.
  **The raw trace stays in the domain layer / content-store; the kernel only ever sees numbers.**

  `{steady_state, response_amplitude, switching_time, separation, oscillation_frequency}`. The

  flat), so it is additive and byte-safe for the existing faces; it becomes load-bearing only for
  the oscillator family. **Ask for B:** if any ingest path pins the secondary key-set, widen it to
  this 5-key set (the derivation `circuit_dynamics.derive_phenotype` is the single source, used by
  BOTH legs, so dry and wet stay identical). A cross-candidate **EC50** phase also exists
  (`circuit_dynamics.ec50_from_curve`) but is derived over a dose ladder, not a single trace, so it
  is a report/analysis-layer quantity, not a per-well secondary channel.

## SEAM 7 (NEW, deepening pass) — thread a per-candidate run CONDITION into the circuit dry leg
**Shared file:** `expos/mcl.py` (`_circuit_candidate_params`).
- **Why:** the cassette/toggle families BAKE their design coord into the graph kinetics at
  construction, so the dry proxy is coord-correct through mcl with no extra param. The new
  **dose-response** family cannot: its design coordinate is the **applied inducer DOSE**, a run
  *condition* that must set the external-input level in the ODE (`simulate(..., input_levels=...)`).
  `_circuit_candidate_params` currently forwards only `{circuit_topology: payload, <var>: level}`
  — no condition — so a dose-response circuit would simulate at a single default dose.
- **Because this pass may not edit mcl, dose-response is kept OUT of the provider claim pool**
  (`provider._circuit_catalogue` = cassette + toggle only, byte-safe for the landed e2e). The
  family is fully built + tested in the library/verify/dynamics/reader and the domain-local e2e
  (where the test passes `coord` explicitly to the adapter).
- **Ask for B (when a dose-response circuit enters the pool):** thread the level's `descriptors`
  coord (or a dedicated `condition` map) into the circuit dry params, and have the adapter map it
  to `input_levels` (the adapter already reads `params['coord']` → `input_levels` when present:
  `circuit_adapter._INDUCER_MAX_LEVEL`). Neutral shape suggestion: forward
  `{... , 'coord': bindings.coords[level]}` for the circuit kind exactly as the wet leg already
  uses `coords`. No kernel change (the dose is a domain-layer condition; kernel still sees numbers).
