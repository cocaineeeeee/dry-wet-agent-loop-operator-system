## M23 reuse (READ-ONLY, un-modified)

`action_ledger.py` / `driver.py` / `recovery.py` were **not touched**. The executor drives
`ActionLedger` purely through its public API: `dispatch` (PENDING-before-I/O), `confirm`
(the sensed gate), `abandon`, hash-chain `verify`/`_replay` on resume, idempotency. The
five faces (normal / wrong-well / timeout / duplicate-reply / partial-resume) are all M23
semantics, exercised — not re-implemented.

**No M23 extension was needed for v0.1.** The one design note for B: mix/incubate/measure
are modelled as **zero-volume self-actions** on the ledger (source==dest, vol 0) so they
pass the transfer-shaped prechecks and share the identical commit gate. If B prefers a
first-class non-transfer action kind in `action_ledger` (rather than the zero-volume
convention), that is the only candidate `action_ledger` change — and it is optional.

## Seams for the integration owner (B, single-writer of mcl / shared schema)

1. **event-kind registration** — every ledger transition already emits a
   `physical_action_transition` event (full-record snapshot). Seam: B registers this event
   kind in the kernel store registry so the M29 dispatch trail lands in `events.jsonl`
   (M29 does not touch the kernel store; payload shape is documented in `action_ledger.py`).
2. **protocol ↔ wet leg** — a screening round should EMIT a `Protocol` from the mcl wet
   experiment (`ExperimentObject.controls` + candidate wells + `bio_readout` needs). Seam:
   an `experiment → Protocol` compiler (analogue of `compile_wet`), reusing
   `protocol_spec.py` / `ot_protocol.py`. Today the protocol is authored domain-locally.
3. **observable / MEASURE → observation** — `ReadPlate`→MEASURE returns a deterministic
   synthetic reading. Seam: bind the committed MEASURE reading to the
   `expression_fluorescence` observable via `timeseries_reader` / `bio_readout`, so a
   dispatched read yields a **trusted observation** entering the existing claim lifecycle
   (DoD #5/#6 cross the seam here — M29 produces the observation, B routes it to
   `claims.py`).
4. **device_ir fingerprint into provenance** — `ir_fingerprint()` exists and is passed as
   the executor's `spec_fingerprint`. Seam: fold it into the experiment `DesignProvenance`
   (alongside `protocol_fingerprint`) so a lowering change flips run identity in the kernel
   provenance chain, not just locally.
5. **volume ledger ↔ deck reconciliation** — the executor seeds reservoir volumes locally;
   a real round should source initial deck state from the shared plate/deck model so the
   M23 `VolumeLedger` double-entry reconciles with the mcl round's declared reagents.
6. **backend registry** — `pick_backend` routes by `Opcode` capability. Seam: a shared
   backend registry so a real instrument (a `SensedState` implementer) can be injected in
   place of the fakes without the protocol/IR changing.
