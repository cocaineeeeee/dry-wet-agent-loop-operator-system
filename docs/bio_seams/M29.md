  ReadPlate/MEASURE read-back to one `expression_fluorescence` `ObservationObject` per
  candidate/control well (fake plate reader `read_well`, per-well). `run_protocol_round()`
  is the whole-round e2e (compile → transactional execute → commit-gated observations).
  The kernel `objects.py` is imported READ-ONLY (never modified); all biology stays in
  `protocols/` + `adapters/physical/`.

## M23 reuse (READ-ONLY, un-modified)

`action_ledger.py` / `driver.py` / `recovery.py` were **not touched**. The executor drives
`ActionLedger` purely through its public API: `dispatch` (PENDING-before-I/O), `confirm`
(the sensed gate), `abandon`, hash-chain `verify`/`_replay` on resume, idempotency. The
five faces (normal / wrong-well / timeout / duplicate-reply / partial-resume) are all M23
semantics, exercised — not re-implemented.

**No M23 extension was needed for v0.1.** The one design note for B: mix/incubate/measure
are modelled as **zero-volume self-actions** on the ledger (source==dest, vol 0) so they
pass the transfer-shaped prechecks and share the identical commit gate. If B prefers a
first-class non-transfer action kind in `action_ledger` (rather than the zero-volume
convention), that is the only candidate `action_ledger` change — and it is optional.

## Seams for the integration owner (B, single-writer of mcl / shared schema)

1. **event-kind registration** — every ledger transition already emits a
   `physical_action_transition` event (full-record snapshot). Seam: B registers this event
   kind in the kernel store registry so the M29 dispatch trail lands in `events.jsonl`
   (M29 does not touch the kernel store; payload shape is documented in `action_ledger.py`).
2. **protocol → ExperimentObject** — ✅ **LANDED** (`protocols/experiment.py`
   `compile_experiment`). Call it to orchestrate an authored `Protocol` as one round: it
   returns a fully-formed `ExperimentObject` (candidates/controls/layout/objective). **B's
   integration point**: fold the returned object into the mcl round exactly as
   `build_experiment` output is folded — its `execution_req.adapter="physical_protocol"`
   + `params={protocol_id, ir_fingerprint}` route it to the M29 physical leg; pass
   `well_designs={well: {construct, coord, …}}` to bind a real construct screen onto the
   candidates (else candidates carry the protocol's own conditions — an honest position/
   replicate screen). The `provenance.protocol_fingerprint` is already stamped (seam #4
   realised on this path). NOTE: the reverse `experiment → Protocol` emitter (analogue of
   `compile_wet`) is a *separate*, still-open direction — this seam is protocol→experiment.
3. **observable / MEASURE → observation** — ✅ **LANDED** (`protocols/experiment.py`
   `bind_measurements`). A **COMMITTED** ReadPlate/MEASURE read-back binds to one
   `expression_fluorescence` `ObservationObject` per candidate/control well (per-well
   `FakePlateReader.read_well`); an **uncommitted** MEASURE yields **zero** observations
   (COMMITTED is the observation-eligibility gate). **Honest trust boundary**: observations
   are emitted `trust=PENDING` — this layer NEVER self-certifies (matches
   `expos.adapters.ingest`, which also emits PENDING; only kernel QC certifies). **B's
   integration point** (DoD #5/#6): route the PENDING observations into the existing QC →
   claim lifecycle (`claims.py`); PENDING→TRUSTED adjudication stays kernel-side. The
   commit gate decides *whether an observation exists*, not its trust.
4. **device_ir fingerprint into provenance** — ✅ **realised on the compile path**:
   `compile_experiment` stamps `ir_fingerprint(lower(protocol))` into
   `DesignProvenance.protocol_fingerprint`, so a lowering change flips the compiled
   experiment's run identity in the kernel provenance chain (test:
   `test_compile_stamps_protocol_fingerprint_and_is_deterministic`). Still open for B: fold
   it into the run-level provenance chain when the round is persisted through the kernel
   store (compile-side is done; kernel-store threading is B's).
5. **volume ledger ↔ deck reconciliation** — the executor seeds reservoir volumes locally;
   a real round should source initial deck state from the shared plate/deck model so the
   M23 `VolumeLedger` double-entry reconciles with the mcl round's declared reagents.
6. **backend registry** — `pick_backend` routes by `Opcode` capability. Seam: a shared
   backend registry so a real instrument (a `SensedState` implementer) can be injected in
   place of the fakes without the protocol/IR changing.
