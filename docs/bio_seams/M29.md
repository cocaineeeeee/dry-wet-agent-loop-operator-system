# M29 protocols / device_ir / physical ‚Äî seam list (v0.1 skeleton)

Team M29 "Execute". Skeleton delivered: `protocols/` (typed device-neutral Protocol:
transfer/incubate/read) + `device_ir/` (Opcode IR + `lower()`) + `expos/adapters/physical/`
(FakeLiquidHandler + FakePlateReader + `dispatch`). Smoke green: `python -m protocols`
(typed protocol ‚Üí 6 device_ir ops ‚Üí fake dispatch ‚Üí all COMMITTED / sensed CONFIRMED).
Reuses M23 `action_ledger` `ActionState` / `SensedOutcome` READ-ONLY. SIMULATION only ‚Äî
no real hardware, no real readings.

Seams for the integration owner (B):

1. **transaction wiring** ‚Äî the fakes today only PRODUCE the sensed bit; they do not drive
   the M23 `ActionLedger`. Seam: wrap `dispatch` in the existing `ActionLedger` so each op
   is `PLANNED‚ÜíPENDING‚Üí(COMMITTED|ROLLED_BACK|AWAITING_RECOVERY)` with the full
   commit/rollback/timeout/duplicate-reply/resume semantics already built in M23. The fake
   backend already speaks the vocabulary; only the ledger dispatch loop is missing.
2. **backend dispatch (capability)** ‚Äî `dispatch` routes by `Opcode` capability. Seam: the
   shared backend registry/protocol so a real instrument (a `SensedState` implementer, cf.
   `run_mcl_loop(physical_backend=...)`) can be injected in place of the fakes without the
   protocol/IR changing.
3. **protocol ‚Üî wet leg** ‚Äî `Protocol` should be derivable from the mcl wet experiment
   (`ExperimentObject.controls` + candidate wells + `bio_readout` needs), so a screening
   round emits a protocol. Seam: an `experiment ‚Üí Protocol` compiler (analogue of
   `compile_wet`), reusing `protocol_spec.py` / `ot_protocol.py`.
4. **observable / MEASURE** ‚Äî `ReadPlate` ‚Üí MEASURE currently returns a synthetic count.
   Seam: bind MEASURE to the `expression_fluorescence` observable read via
   `timeseries_reader` / `bio_readout`, so a dispatched read yields a real observation.
5. **device_ir fingerprint** ‚Äî the lowering is deterministic; the IR + opcode-set version
   should enter provenance so a protocol's compiled form is auditable and a lowering change
   flips run identity.
6. **volume/deck state** ‚Äî a Transfer's volume should reconcile with the M23 `VolumeLedger`
   (double-entry deck accounting) rather than the placeholder no-op here.
