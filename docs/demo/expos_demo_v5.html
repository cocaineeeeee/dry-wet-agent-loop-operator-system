05 · Honesty ledger

The six criteria for the first biological loop—decisive on the raw path, caveats kept

①a construct never fakes a molecular geometry✅ by design — Contract v3
②phenotype data truly changes a claim — three-state separation on one claim head✅ high→supported · flipped→rejected · flat→insufficient
③the changed claim changes the knowledge state✅ fingerprint d04b4d05 → 7f8ce457
④the changed knowledge changes the next round's construct✅ mechanism-proven · ⚠️ not self-triggered in-loop
⑤all three faces (high / flipped / flat) hold in a running loop✅ in the running loop · 12/12 tests
⑥kernel / ledger / certification stay biology-blind✅ countersigned — 0 biology literals
The biological loop closes decisively and is double-signed (M24-B) — three-state separation on one claim head (expression_high → supported, e-product 102→1033, +0.234; expression_flipped → rejected, e=42, −0.004; flat → insufficient, e=0, p=1.0) plus a knowledge-fingerprint migration; the percent-of-control (controls) path now certifies too via a scale-aware w_min (effective 83.33 on the percent scale). Proposal reordering (④) stays mechanism-proven, not self-triggered inside a single low-signal loop — the honest caveat that survives the double-sign.
