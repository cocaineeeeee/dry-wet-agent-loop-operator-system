# 深度思考 —— expos 的真实贡献、有效性威胁与路线修正

> 2026-07-10，M4 完成之际的一次全局审视。输入：四轮调研（REFERENCE_MAP §1–§12）、
> M0–M4 实现与 12 轮 agent 把关暴露的模式。这不是进度报告，是自我批判。
> 结论已按"必须改 / 应该改 / 记入 backlog"分级；BUILD_PLAN 的修改提案见 §5，待批准后落。

---

## 1. 这个项目真正的贡献是什么（收窄主张）

四轮调研后可以收窄为两条**站得住的**差异化主张，其余都是 commodity：

1. **方法学空白（已两次独立核查）**：没有任何公开基准在闭环优化对比中注入*结构化*系统偏差
   （空间场/漂移/批次）——现有工作止步于 iid 噪声或通用 y-腐蚀。我们的"模拟器 + 六注入器 +
   naive vs OS 对比"落在这个空白上。
2. **provenance 驱动的失败归因作为一等内核服务**：最接近的先例 A-Lab（ARROWS3 失败链）把
   合成失败与表征失败混在一条决策链上、缺独立复核——正是它被质疑之处。expos 把
   "测量不可信"与"参数不可行"分类路由，是对该缺陷的直接回应。

推论：**M5+M6（QC/归因）和 M9（对比实验）是项目的心脏**；M4 的 GP/BO 有意保持 commodity
（这是对的——评审者不会因为我们的 BO 更花哨而信任结论，只会因为对比实验更诚实而信任）。

## 2. 有效性威胁（诚实清单——评审者会打的三枪）

### 威胁 A：既当运动员又当裁判（demo 自证循环）
我们**设计了伪影，又设计了恰好能抓它们的 QC**。edge_evaporation 抬高边缘读数，QC 里就有
边缘效应检查——怀疑者一句话就能击穿："你为你注入的每种腐蚀各造了一个探测器。"

**对策（必须进 M9）**：
- **留出伪影（held-out artifact）**：评估时加入一种签名库*没有*对应假设的注入器
  （建议：低阶空间相关高斯随机场偏差——六个签名都不直接匹配它），报告 QC 对
  未见过的腐蚀类型的行为（能兜住多少、错判成什么）；
- **幅度敏感性扫描到失效边界**：不只报 QC 赢的幅度档（25–50% 边缘增强对 4% 噪声是送分题），
  要扫到 5–10% 的小伪影区，**画出检出率-幅度曲线与失效点**——展示边界比展示胜利更可信；
- **标定/评估场景分离**：QC 阈值（0.3/0.6、SBB 映射、控制带 k）在场景集 A 上标定，
  在场景集 B（不同种子 + 不同伪影参数）上评估，杜绝阈值过拟合。

### 威胁 B：naive 是稻草人（最重要的实验设计缺陷）
naive vs OS 的二臂对比不够。一个**鲁棒但不做信任路由**的基线（副本中位数聚合 +
稳健 GP/温莎化 y）能吸收大量伪影伤害而完全不需要 QC。如果 OS 只赢 naive 而赢不了
robust-blind，"信任路由"的增量价值就没被证明。

**对策（必须进 M9）**：对比升级为**三臂**——
1. `naive`：全信（现状）；
2. `robust-blind`：副本中位数 + 稳健损失，无 QC 无路由（信任盲的工程上限）；
3. `os`：完整信任路由 + 失败模型 + 失败感知规划。
预期的诚实结果：robust-blind 在対称/孤立伪影（眩光/灰尘）上接近 os；os 的增量价值集中在
**结构性偏差**（边缘/梯度/批次——中位数无法救，因为整组副本同向偏移）与
**失败感知的下轮设计**（复测/歧义消解动作）。这个 nuance 正是论文级结论。

### 威胁 C：QC 税（干净数据上的假阳性代价）
一个把 20% 好数据错误隔离的系统能在含伪影基准上"赢"，实际却不可用。

**对策（必须进 M9）**：加**零伪影对照场景**——无注入器时 os 相对 naive 的效率损失
（被隔离的好观测比例、最终 regret 差、多花的复测预算）作为一等指标报告。QC 税 ≤5%
应作为 M5 验收线之一。

## 3. 架构层的三个深层观察

### 3.1 信任是二值的，证据是连续的——per-point alpha 是原则性的桥
现状：suspicion 连续分 → 硬阈值 → 三桶（TRUSTED/SUSPECT/FAILED），响应模型全权重吃
TRUSTED。阈值附近的观测（0.29 vs 0.31）命运截然不同——脆。
GP 已预留逐点 alpha（§11.1：副本方差 → alpha）。原则性演进：
- **近期（M5 可顺手做）**：TRUSTED 观测的 alpha 按副本方差设（已规划）；
- **中期（backlog）**：alpha 再乘 (1 + g(suspicion))——低嫌疑连续降权替代部分硬切；
- **远期（backlog，最漂亮的一步）**：**偏差校正复归**——归因给出 edge_evaporation 及
  估计增益后，把被隔离观测*去偏还原*、以膨胀的不确定度（大 alpha）复归训练集。
  失败模型由此升格为**测量误差模型**，"失败是资产"从口号变成字面事实。
  这也是 QUARANTINE 数据不再是死资本的路径。

### 3.2 公平性主张依赖 mode 的结构分离（M5 重构红线）
"naive 与 os 共享其余全部代码"是对比公平性的根基。现状 `_route_naive` 内联在 loop 主体，
M5 接 QC 时若长出 `if mode == ...` 分叉树，公平性主张就名存实亡。
**M5 落地时**：把"裁决策略"抽成单一注入点（naive 策略=常量 TRUSTED；os 策略=QC 裁决），
loop 主体零分支。这不是风格洁癖——是论文方法节的一句话："两臂仅在裁决策略对象上不同。"

### 3.3 统计功效的现实检查
5 哨兵/轮 × ≤8 轮、副本 n=2，这是**极小样本体制**——调研结论（§11.4）已经给出正确姿势
（效应量+跨轮累积、置换检验、前 2–3 轮控制图只记不判），但要预期：**单轮内小幅伪影
（<10%）大概率抓不到，跨轮累积才抓得到**。这必须如实写进 M9 结果叙述，
并且正是"检出率-幅度曲线"要展示的内容。demo 场景（50% 边缘增强）是给观众看的；
敏感性扫描是给评审者看的。两者都要。

## 4. 过程层反思（agent 把关的模式总结）

12 轮 agent 把关抓到的真问题几乎全落在三类：**信息泄漏通道**（coating oracle、无噪声
secondary）、**语义混叠**（批次-捕获序、批次效应 vs 漂移不可辨识）、**静默行为**
（约束拼错放行、重复 well_id 透传、越界静默返回）。这提示 M5/M6 的自查清单应以这三类
为纲——QC 层本身最容易犯的恰是第一类（拿了不该拿的证据）与第三类（该报警时沉默）。
另：对抗审查（找违背）+ 压力测试（实跑找崩）+ 合规比对（找缺漏）三路正交、互不替代，
这个配置保留到 M10。

## 5. 对 BUILD_PLAN 的修改提案（待批准）

1. **M9 验收标准升级**：二臂 → **三臂**（+robust-blind）；新增四项必报指标——
   零伪影 QC 税、留出伪影行为、检出率-幅度曲线（Slurm 扫描）、标定/评估场景分离声明；
2. **M5 验收线追加**：干净场景假阳性率（QC 税）≤5%；裁决策略抽成单一注入点
   （loop 主体零 mode 分支）；
3. **新增 backlog 节（M10 后）**：偏差校正复归（per-point alpha 通道）、连续信任降权、
   轮内可重入检查点（Bluesky 式）、OpenLineage 导出器、BayBE 可选后端。

## 6. 一句话收束

expos 的价值不在"能抓伪影"（那是自证循环），而在**把"测量可信度"变成实验 OS 的
一等内核语义，并用一个诚实的三臂基准证明：当伪影是结构性的时候，信任路由的增量
价值无法被鲁棒统计替代**。M5–M9 的每个决定都应该回答同一个问题：
"这让那条主张更可证伪，还是更不可证伪？"
